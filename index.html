MVC, MVP, MVVM, MVI

These are design patterns used to organize and manage the codebase. They help separate concerns, improve maintainability, and facilitate testing:

  • MVC: Model-View-Controller, where logic and UI are separated into different components.
  • MVP: Model-View-Presenter, which separates the view logic from the business logic.
  • MVVM: Model-View-ViewModel, widely used in Android with data binding.
  • MVI: Model-View-Intent, focusing on unidirectional data flow.
Learn more about Architecture Patterns

Repository Pattern

Dependency Injection (Dagger, Hilt, Koin)

Dependency Injection (DI) is a design pattern that provides dependencies to objects. Popular DI libraries in Android include:

  • Dagger: A robust DI framework with compile-time validation.
  • Hilt: Built on Dagger, designed specifically for Android to simplify DI implementation.
  • Koin: A lightweight and simple DI framework for Kotlin.
